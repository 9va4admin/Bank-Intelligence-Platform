Presentee Bank · Outward Clearing

Presentee Pipeline — Your customer deposits a cheque drawn on another bank

Bank acts as presenting bank. Flow: scanner → MICR → CTS-2010 compliance → lot → endorsement → NGCH submission → session reconciliation.

1
Physical Scanner Capture
OEM adapter (Panini / Canon / Generic) normalises raw scan into ScanResult. Captures: front DPI, rear DPI, front/rear file size KB, colour depth, MICR raw string. Duplicate scan_id guard enforced by ScanSessionManager.
scanner/adapters.py · scanner/session.py · scanner/models.py
Module ✓
2
MICR Line Extraction & Parsing
E-13B encoded MICR band parsed: routing number (between ⑆ symbols), cheque number (between ⑆ and ⑈), account number (between ⑈ and ⑉). PII rule: only last 4 digits of account stored — never the full number. Invalid/unparseable MICR → instrument rejected at scan time.
scanner/micr.py
Module ✓
3
CTS-2010 Image Compliance Validation
Per RBI CTS-2010 standard (immutable — not bank-configurable): front DPI ≥ 200 · colour depth ≥ 24-bit RGB · rear may be 8-bit grayscale · file size ≤ 50 KB per side · IQA score ≥ 0.70 · MICR band score ≥ 0.80 · both front and rear images mandatory. Compliance certificate generated as XML (CTS2010_CERT_{IFSC}_{date}_{lot}.xml).
compliance/cts2010.py · compliance/exporter.py
Module ✓
4
Lot Creation & Instrument Assignment
Instruments grouped into lots (LOT_{IFSC}_{date}_{session}_{NN}). Max 200 instruments per lot (NGCH limit, configurable). Auto-assignment fills current open lot before creating a new one. Sub-member MICR prefix → instrument routed to sponsor lot via SMBForwardingWorkflow.
lot/manager.py · sub_member/router.py
Module ✓
5
Endorsement Stamping
Bank IFSC, presentation date, lot reference, account suffix (last 4 digits only) stamped onto rear image. QR code encodes: ASTRA-ENDORSE|{IFSC}|{instrument_id}|{acct_suffix}|{date}. Template expiry validated before stamping.
endorsement/stamper.py · endorsement/models.py
Module ✓
6
NGCH Submission & Acknowledgement Validation
NGCH adapter (MCP server) submits lot. Exactly-once: UUIDv7 idempotency key per instrument prevents duplicate submissions. NGCH acknowledgement parsed and verified — submission not complete until NGCH confirms. Currently SFTP-based; REST and MCP upgrade paths defined in NPCI Readiness Plan (Section 17).
mcp/ngch_adapter.py · workflows/activities/ngch_filer.py
Module ✓
7
Session Reconciliation (NGCH vs CBS)
After clearing session closes: NGCH-filed records crossed against CBS posting confirmations. Per-instrument status: MATCHED · NGCH_ONLY (filed, no CBS posting) · CBS_ONLY (CBS posted, no NGCH record) · AMOUNT_MISMATCH · PENDING (in-flight). RRF (Return Reason File) generated for returned instruments.
reconciliation/engine.py · rrf/generator.py
Module ✓

Drawee Bank · Inward Clearing

Drawee Pipeline — Cheques drawn on your bank arrive from NGCH

One AI agent per cheque. All validations must complete in < 600ms wall clock. IET breach rate target: 0.000%.

0
IET Watchdog — Spawned First (Always)
IETWatchdogWorkflow spawned as FIRST child before any activity. parent_close_policy = ABANDON — watchdog survives parent failure. At T−30 seconds: emergency-files CONFIRM to NGCH via ngch_adapter. Structural guarantee — not configurable.
workflows/iet_watchdog_workflow.py
Done ✓
1
OCR Field Extraction (GOT-OCR2.0)
Extracts: MICR line, amount in figures, amount in words, date, payee name, drawer name. Per-field confidence checked against config_service.get("ocr.min_confidence") — low confidence → HUMAN_REVIEW. vLLM unavailable → HUMAN_REVIEW (degraded, never crashes workflow). MICR line used for SMB routing identification (DIRECT vs SUB_MEMBER tag).
workflows/activities/ocr.py · workflows/activities/amount_words_parser.py

2
Alteration Detection (Qwen2-VL Cascade + Kill Switch)
6 detection layers: field content anomaly · ink physics (pen vs correction fluid) · paper-fibre distortion · correction fluid spectral signature · chemical alteration stain · overwriting brightness differential. L1/L2 cascade (Gemini Fix A): L1 = Qwen2-VL 7B (~100ms); escalate to L2 = Qwen2-VL 72B if confidence < threshold OR high-value OR OPA policy override. Kill Switch checkpoint 1 (KP/KC): if active, Vision AI skipped and flag set for decision backstop. Alteration detected → immediate STP_RETURN (hard gate).
workflows/activities/alteration.py · kill_switch/vision_ai_kill_switch.py
Done ✓
3
Signature Verification (Siamese Network)
Specimen retrieved from Signature Vault (Redis, hashed key: sig:{bank_id}:{sha256(account)}). Siamese neural network compares presented signature against all vault specimens; best_match_score used. Vault miss (no specimens on file) → HUMAN_REVIEW — invariant, never auto-return. Match score < config_service.get("signature.min_match_score") → HUMAN_REVIEW. Model unavailable → HUMAN_REVIEW (degraded).
workflows/activities/signature.py · vaults/signature_vault.py
Done ✓
4
PPS (Positive Pay System) Verification
Vault lookup by (account_number, bank_id, cheque_number). If registered: amount cross-check (±₹1 tolerance) AND payee name comparison (case-insensitive). PPS vault miss → HUMAN_REVIEW (not auto-return — vault-miss invariant). Amount mismatch → HUMAN_REVIEW. Payee mismatch → HUMAN_REVIEW.
workflows/activities/pps.py · vaults/pps_vault.py
Done ✓
5
CBS Account Status + Balance Check
Account status routing: FROZEN / CLOSED / NPA → STP_RETURN (hard gate, immediate). DORMANT → HUMAN_REVIEW (ambiguous). AccountNotFoundError → STP_RETURN. ACTIVE → PROCEED with available balance. CBS unreachable (CBSUnavailableError) → CBS_UNAVAILABLE (degraded, image-only path continues). Available balance passed to fraud activity for feature engineering.
workflows/activities/cbs.py · shared/cbs_connector/
Done ✓
5b
Stop Payment Instruction Check
Implemented (July 2026). Two-stage check: (1) Bloom filter pre-check (BF.EXISTS) — Bloom hit → HUMAN_REVIEW (never STP_RETURN, due to probabilistic false-positive rate). (2) CBS authoritative lookup — confirmed stop → STP_RETURN. CBS unavailable → HUMAN_REVIEW (never crashes). Wired into cheque_workflow.py after CBS balance check.
workflows/activities/stop_payment.py · cheque_workflow.py — 22 tests
Done ✓
6
Fraud Scoring (XGBoost + SHAP + Optional LLM Rationale)
XGBoost ensemble scores: amount, OCR confidence, alteration flag, CBS status, PPS result, signature score. SHAP values mandatory in every result — required for audit compliance before NGCH filing. Model unavailable → rule-based fallback (alteration +0.60, low OCR +0.15, very high amount +0.05) — SHAP still populated from rule contributions. Optional: Llama 3.3 70B (queue: cts-reasoning) synthesises 3–5 sentence human-readable rationale from SHAP top-5 + full upstream context, using Headroom compression (~75–82% token reduction).
workflows/activities/fraud.py
Done ✓
6b
OPA Business Policy Evaluation (Layer 4)
Implemented (July 2026). OPAClient in shared/opa_client.py calls POST /v1/data/astra/cts/routing. Gate is injected into synthesise_decision() after the kill-switch backstop. OPA → HUMAN_REVIEW or AUTO_RETURN short-circuits all downstream gates. Government cheques, court orders, high-value first-clearing-day rules are now enforced at runtime. OPA unavailable → safe default PROCEED (existing numeric gates still apply). 21 tests.
shared/opa_client.py · workflows/activities/decision.py · infra/opa/policies/cts_routing.rego
Done ✓
7
Decision Synthesis — Priority Gates
Kill Switch backstop (checkpoint 2) evaluated first — catches mid-flight race (kill switch activated during the 120-second Qwen2-VL call). Then hard gates in order: CBS RETURN → STP_RETURN · Alteration detected → STP_RETURN · Fraud score ≥ threshold → HUMAN_REVIEW · OCR confidence < minimum → HUMAN_REVIEW · Signature match < minimum → HUMAN_REVIEW · CBS unavailable → HUMAN_REVIEW · PPS miss → HUMAN_REVIEW. STP_CONFIRM only when: all gates clear AND combined_confidence (OCR×0.5 + sig×0.5) ≥ stp_auto_confirm_threshold. All thresholds from config_service — never hardcoded.
workflows/activities/decision.py
Done ✓
8
NGCH Filing + Audit Write
file_to_ngch is the ONLY place in ASTRA that calls NGCH. Exactly-once: workflow_id is the idempotency key. DuplicateFilingError is non-retryable (prevents double-pay). Decision published to cts.decisions.{bank_id} Kafka topic after filing. Audit event written to Immudb (HSM-signed) for every terminal state.
workflows/activities/ngch_filer.py · workflows/activities/write_audit.py

NOTE · CTS-2010 Inward Re-Validation Not a Workflow Step
CTS-2010 standard constants and the compliance certificate exporter exist for the Presentee path. On the Drawee side, NGCH is supposed to deliver only compliant images, but there is no step in cheque_workflow.py that re-validates the received image against these standards before processing. This may be acceptable (trusting NGCH pre-filter) but is worth a deliberate decision.
