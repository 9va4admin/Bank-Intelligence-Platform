Architecture

Locked Decisions

These decisions are final as of this deep-dive session. No code should be written that contradicts them.

DecisionWhat was settled
One binaryConfiguration-driven deployment. Same codebase for all scenarios. RBI compliance certified once. Features gate via Helm Layer 2.
Branch → PU mappingBase entity, mandatory in all scenarios. Administrative decision by SB — not geographic. A Pune branch may map to Mumbai PU.
ProcessingUnitFirst-class entity. All Kafka topics and Temporal task queues are PU-scoped, not just bank-scoped.
Command CenterLogical layer for SB (thin, within existing workflows). New microservice for Agency (substantive routing across multiple SBs).
Session cut-offNPCI-mandated and uniform across all banks. No per-SB cut-off. Agency's effective cut-off = NPCI cut-off − per-SB submission buffer (config).
Clearing session granularityOne clearing session per SB (groups all SMBs routing through that SB).
OCR eliminatedScanner hardware handles MICR (magnetic head) and field OCR. GOT-OCR2.0 is removed entirely. Vision LLM validates, does not extract.
Vision ordering — PresentmentVision LLM is the LAST step before lot seal. Scanner → CTS-2010 → Lot → Vision → seal or HOLD.
Vision ordering — DraweeVision LLM is FIRST. CBS is LAST. Fail fast before expensive external calls.
Human Review — DraweeAlways routed to SMB, regardless of where Vision LLM ran. SMB reviewer owns the decision.
Signature Vault locationAt SMB level when SMB has own CBS. At Agency level when Agency manages CBS. Always co-located with CBS.
SMB with own CBSFull ASTRA deployed at SMB. Vision LLM optional (depends on SMB GPU capability). If no Vision LLM at SMB — Vision runs at PU, results returned.
EEHNew microservice. Browser-based (gRPC-Web + Envoy proxy). Separate from API Gateway. Serves SMB/external branches.
Branch clientBrowser-based. Requires local scanner bridge agent (tray app on branch workstation) to bridge browser ↔ hardware scanner.
Mismatch resolutionPU status: HOLD. Branch sees held queue in UI. Actions: Go-Ahead (proceed) or Rejected (discard). Branch decides based on physical examination.
CTS Miniature DBStop Payment and PPS live in ASTRA's own DB (synced from CBS). CBS live calls are only: get_balance, get_account_status, get_signatures.
Deployment

Three Deployment Scenarios

Scenario 1
SB + SMB — SMB CBS on-prem
  • ASTRA deployed at SB
  • ASTRA also deployed at SMB (CBS concern)
  • CBS proxy MCP agent at SMB
  • Signature vault at SMB level
  • Vision LLM at SB PU (if SMB has no GPU)
  • Human review queue at SMB portal
  • CC = logical layer → NPCI
  • deployment_mode: SB_NGCH
Scenario 2
Agency + SMB — Managed CBS
  • ASTRA at Agency only
  • Agency manages CBS for all SMBs
  • Signature vault at Agency level
  • Vision LLM at Agency PU
  • CC = new microservice → multiple SBs
  • One SB connector per SB relationship
  • One clearing session per SB
  • deployment_mode: AGENCY_SB_RELAY
Scenario 3
Agency + SMB — SMB CBS on-prem
  • ASTRA at Agency + at each SMB
  • CBS proxy MCP agent at each SMB
  • Signature vault at each SMB level
  • Vision LLM optional at SMB
  • CC = new microservice → multiple SBs
  • Most complex; all isolation enforced
  • deployment_mode: AGENCY_SB_RELAY
  • smb_cbs_proxy.enabled: true
Implication: Scenarios 2 and 3 share the same deployment_mode: AGENCY_SB_RELAY. The difference is smb_cbs_proxy.enabled. The agency-cc-service microservice is active in both.
Processing

Processing Sequences (Settled)

Outward — Presentment (Vision LAST)

Scanner hardware MICR (magnetic head) — most reliable, no AI needed Built-in OCR → amount, payee, date, cheque serial Images → color front, grayscale front, rear ↓ upload to EEH via gRPC streaming CTS-2010 compliance check ← image format, DPI, dimensions FAIL → reject immediately, notify branch ↓ Lot assignment ← PU-level, branch → PU via CRL ↓ Vision LLM ← LAST validate scanner data vs image, figures vs words, alteration MATCH → lot sealed → endorsement → NGCH / SB MISMATCH → PU status: HOLD → push to branch queue Branch: Go-Ahead (proceed) | Rejected (discard)

Inward — Drawee (Vision FIRST, CBS LAST)

NGCH → SB → Command Center → PU ↓ Vision LLM ← FIRST early filter — alteration? image clear? readable? FAIL → STP_RETURN immediately, no further processing ↓ CTS checks MICR validation, amount cross-check, CTS-2010 ↓ Stop Payment check ← ASTRA miniature DB (Redis vault) ↓ PPS check ← ASTRA miniature DB (Redis vault) ↓ Signature Verify ← vault at SMB level or Agency level (Redis) ↓ Fraud Score ← XGBoost — synthesises all signals above + SHAP ↓ CBS ← LAST get_balance, get_account_status — most expensive, earned by passing all above ↓ STP_CONFIRM | STP_RETURN | HUMAN_REVIEW → SMB portal always

NEW ScannerConfig

Branch-level configuration that tells ASTRA how to parse the OEM scanner software's drop-folder output. Every scanner OEM produces a different metadata file format (CSV dialect, XML, fixed-width) with different field names. This config is the mapper — the rest of the pipeline is OEM-blind once mapping is applied.

cts.scanner_configs
  scanner_config_id   UUID PRIMARY KEY
  bank_id             TEXT NOT NULL
  branch_id           UUID REFERENCES cts.branches   -- null = bank-wide default
  scanner_oem         TEXT NOT NULL            -- PANINI | DIGITAL_CHECK | MAGTEK | RDM | OPEX | GENERIC
  scanner_model       TEXT NOT NULL            -- e.g. "Panini Vision X", "TS240"
  output_format       TEXT NOT NULL            -- CSV_COMMA | CSV_PIPE | CSV_TAB | XML | FIXED_WIDTH
  date_format         TEXT NOT NULL            -- strptime pattern, e.g. "%%d%%m%%Y"
  amount_format       TEXT NOT NULL            -- DECIMAL_DOT | DECIMAL_COMMA | INTEGER_PAISE
  field_mapping       JSONB NOT NULL           -- OEM field name → canonical field name
  image_naming_pattern TEXT NOT NULL           -- e.g. "{batch_id}_{seq:04d}_{side}.tif"
  image_side_mapping  JSONB NOT NULL           -- OEM side codes → color_front | grey_front | rear
  drop_folder_path    TEXT NOT NULL            -- local path or sftp://host/path
  is_active           BOOLEAN DEFAULT true
  created_at          TIMESTAMPTZ DEFAULT now()
Pattern identical to EJ OEM fingerprinting: EJ uses dynamic detection (fingerprint the file). Scanner uses explicit config per branch (bank knows their hardware). Both converge to a canonical model — the downstream pipeline is OEM-blind in both cases.

Lightweight tray/background application installed on the branch workstation. Bridges browser (which cannot access USB/TWAIN hardware) to the physical scanner. Exposes a local WebSocket on localhost:7431 that the branch web app connects to.

Protocol

gRPC Service Definitions

NEW EEHService

service EEHService {
  // Branch uploads cheques one-by-one; server acks each in real-time
  rpc UploadCheque(stream ChequePayload) returns (stream ChequeAck);

  // Supervisor seals the lot when scanning is done
  rpc SealLot(LotSealRequest) returns (LotSealAck);

  // Branch polls its held/mismatched items queue
  rpc GetMismatchQueue(BranchQueueRequest) returns (stream MismatchItem);

  // Branch resolves a held item: GO_AHEAD or REJECTED
  rpc ResolveMismatch(MismatchResolution) returns (ResolutionAck);

  // Session status (for branch dashboard)
  rpc GetSessionStatus(SessionStatusRequest) returns (SessionStatus);
}

message ChequePayload {
  string branch_id      = 1;
  string session_id     = 2;
  string idempotency_key = 3;   // branch_id + session_id + seq_no
  string micr_line      = 4;    // from scanner
  string amount_figures = 5;    // from scanner OCR
  string amount_words   = 6;    // from scanner OCR
  string payee_display  = 7;    // masked
  bytes  image_color    = 8;    // TIFF front color
  bytes  image_grey     = 9;    // TIFF front grayscale
  bytes  image_rear     = 10;   // TIFF rear / endorsement
  string scan_timestamp = 11;   // ISO-8601
}

NEW SBConnectorService (Agency → SB)

service SBConnectorService {
  // Submit grouped lot to SB platform
  rpc SubmitLot(LotSubmissionRequest) returns (SubmissionAck);

  // Receive inward instruments SB forwards to agency
  rpc PollInward(PollInwardRequest) returns (stream InwardInstrument);

  // Receive returns and RRF after session
  rpc PollReturns(PollReturnsRequest) returns (stream ReturnItem);

  // Get settlement position for agency's SMBs
  rpc GetSettlement(SettlementRequest) returns (SettlementReport);
}

NEW CRLService (internal)

service CRLService {
  // Resolve IFSC → PU (used by EEH, IEH, ChequeProcessingWorkflow)
  rpc ResolveBranch(ResolveRequest) returns (BranchResolution);

  // Invalidate cache on branch remapping
  rpc InvalidateCache(InvalidateCacheRequest) returns (InvalidateCacheAck);
}

Security

Security Additions

ComponentSecurity requirement
EEH — Branch identitymTLS client certificate per branch, issued by SB's internal CA. Certificate encodes branch_id, bank_id, validity. Revocation via CRL distributed to EEH on every connect.
EEH — Operator identityOperator JWT via SAML (branch IdP). Scoped: branch_id, operator_id, role, session_id, pu_id. 1-hour expiry + refresh. Separate from branch mTLS cert.
EEH — Request integrityHMAC-SHA256 per gRPC message. Key from branch's local Vault agent. Header: X-ASTRA-Sig: {timestamp}.{hex(hmac)}. ±5 min window for replay protection.
Network — EEH exposureEEH only accessible via MPLS or IPSec/WireGuard VPN. Not on public internet. Istio Ingress Gateway in DMZ. Source IP allowlist per branch (static IPs or VPN pool).
Scanner bridgeBridge listens on localhost:7431 only. Signed binary (code signing certificate). Communicates only with the ASTRA branch portal origin (CORS).
SMB CBS proxyGo binary connects outbound (SMB never opens inbound). mTLS. Token scoped to: smb_id, allowed_tools only. Every call logged at SMB side. SMB can revoke token from their admin.
Agency SB connectorCredentials in Vault: secret/astra/{agency_id}/sb/{sb_id}/. Separate key slot per SB. SFTP: RSA-4096 key. API: API key + HMAC. Rotated per SB's policy.
Branch cert lifecycleEEH_CERT_EXPIRY_WARN message 30 days before expiry. CRITICAL at 7 days. Expired cert = EEH connection refused. Renewal process in branch onboarding runbook.
